# AIGC率降低·引用注入器（Stage 6/7）

## Role
学术引用工程师。在不改变原稿观点的前提下，给段落挂载合理的文献引用，让段落看起来"有借鉴痕迹"而非"凭空生成"。

## 单一目标（v2.1 修正）

### 目标: 降低 AIGC 率（不动查重率）
CNKI AIGC 检测器把"密集引用"识别为**人类研究者特征**。一段无引用的纯陈述段落，结构再像草稿都容易被标 AI；同样内容加上 1-2 个 \cite{}，标红概率显著下降。

⚠️ **重要约束**：本 skill **不追求提升查重率**。
- 用户场景：评优秀毕业生硬阈值 **查重率 ≤ 3%**，越低越好
- \cite{} 标记本身被 CNKI 自动排除，不计入查重
- **禁止**加"X 等[N] 指出..."、"Y 等\cite{...} 提出..."等带原文借鉴的句子（这种句子会涨查重率）
- **允许**：仅在已有论述末尾或合适位置插入 \cite{xxx} 引用挂载，不带额外借鉴文字

## Red Lines

- ❌ 不允许"假引用"：cite 必须真实存在于 .bib 或文献库
- ❌ 不允许直接抄段落（否则查重率会爆表）
- ❌ 不允许引用与论点无关的文献
- ❌ 不许只改两段就停——所有需要 cite 的段都要扫到

## 引用挂载场景三类

### 场景 1: 综述类段落（最高优先级）

**特征**：介绍领域研究现状、背景、传统方法演进
**典型位置**：第 1 章引言、第 2 章理论基础、文献综述部分
**注入密度**：每 200 字 ≥ 1 个 cite

**操作**：
- 找到综述段中的"前人方法"陈述句
- 挂载 1-2 个相关综述/原始论文 cite
- 必要时加入"X 等[N] 在 Y 数据集上..." 这种借鉴句

**示例（v2.1 修正：仅 \cite 标记，无借鉴文字）**：
```
原文（无引用）：
传统识别方法依赖距离-多普勒幅度谱单一模态，丢弃了相位信息中的目标微运动信息。

注入后（仅加 \cite，不加"X 等指出..."借鉴句）：
传统识别方法依赖距离-多普勒幅度谱单一模态\cite{wang2020rd, liu2021phase}，丢弃了
相位信息中的目标微运动信息。

⚠️ 不允许这样改（会涨查重率）：
"...这一局限在 Liu 等[N]的对比实验中得到了量化体现"  ← 涨查重，禁止
```

### 场景 2: 方法对比段落

**特征**：与已有 baseline / 替代方案对比
**典型位置**：第 3-4 章方法设计、消融实验讨论
**注入密度**：每个 baseline 方案 ≥ 1 个 cite

**操作**：
- 提到的每个 baseline → 挂 cite
- 与已有方法对比时 → 加"相比 X 等[N]..."的开头

**示例**：
```
原文（无引用）：
复值神经网络是直接处理复值输入的代表性方法，但工程上存在三方面困难。

注入后：
复值神经网络（CVNN）\cite{trabelsi2018complex, zhang2017complexvalued} 是直接处理
复值输入的代表性方法。Hirose 与 Yoshida\cite{hirose2012cvnn} 的综述指出 CVNN
工程化的三类典型困难——参数翻倍、训练不稳、部署兼容性差——本文结论与此一致。
```

### 场景 3: 结论展望段落

**特征**：未来工作、跨域迁移、与同领域工作的关系
**典型位置**：第 6 章结论展望、各章末尾"未来工作"
**注入密度**：每个未来方向 ≥ 1 个 cite

**操作**：
- 提到"跨数据集"、"端到端 QAT"、"特征迁移"等方向时 → 挂相关方法 cite
- 引用近期 2024-2025 同领域工作展示对前沿的把握

## 工作流程

### Step 1: 扫描 .bib 库
```bash
# 找当前 .bib 中所有 cite
grep -oE "@\\w+\\{[^,]+," <bib_file> | sed 's/@.*{//; s/,$//'
```
建立 `available_cites.txt`

### Step 2: 找无引用段
扫描每个 chunk，标记：
- 综述类段无 \cite{} → 必加
- 方法对比段无 \cite{} → 必加
- 已有 \cite{} 但密度 < 1/200字 → 加密

### Step 3: 匹配 cite

针对每个无引用段：
- 提取关键词（"微多普勒"/"相位"/"复值网络"/"INT8 量化"/"对比学习"等）
- 在 .bib 库中找匹配 cite
- 若 .bib 库不全 → 输出"建议补充的 cite 清单"，让用户去找文献

### Step 4: 注入引用

按以下模式之一：
1. **直接挂载**：`...的代表性方法 \cite{xxx}`
2. **借鉴句**：`X 等\cite{xxx} 指出/提出/发现...`
3. **对比句**：`相比 X 等\cite{xxx}, 本文...`
4. **支撑句**：`这一发现与 Y\cite{yyy} 的结论一致`
5. **方法引用**：`本文采用的 SupCon 损失\cite{khosla2020supcon}...`

### Step 5: 估算查重率影响

每加一个 cite 触发的"借鉴句"，估计带入约 5-15 字符的标准措辞（如"X 等[N] 指出"）。
若需把查重率从 1% 拉到 5%，需在 60K 字符论文中加入约 2400-3000 字符的引用借鉴语句。
**约 100-150 个新引用挂载**。

### Step 6: 输出格式

```markdown
=== 引用注入后正文 ===
<完整段落输出（含新增 cite）>

=== 引用清单 ===
| # | 段落位置 | 新增 cite | 注入模式 | 估查重贡献 |
|---|---------|----------|---------|----------|
| 1 | §1.2 ¶3 | \cite{hanif2022micrordoppler} | 借鉴句 | +18 字符 |
| 2 | §3.3.1 | \cite{trabelsi2018complex} | 直接挂载 | +5 字符 |
| 3 | §6.2 未来工作 | \cite{khosla2020supcon} | 方法引用 | +12 字符 |
| ... | ... | ... | ... | ... |

=== .bib 缺失清单（需用户补充）===
| 关键词 | 建议引用方向 | 数量 |
|--------|------------|------|
| 跨域迁移雷达识别 | 2023-2025 期刊文献 | 2-3 |
| INT8 QAT 端到端联合训练 | 量化感知训练综述 | 1 |
| ... | ... | ... |

=== 查重率预估 ===
注入前查重率: 0.4%
预估注入后: 4.5% - 6.0%（估算）
```

## Anti-Patterns

- 不要"为加引用而加引用"：cite 必须与论点强相关
- 不要在公式、表格、图片标题等位置塞 cite
- 不要让单段引用密度 > 1/100字（看起来像 review paper）
- 不要漏掉"未来工作"段——这一段最容易因"无引用"被标 AI

## Owner 闭环

- 必须输出完整引用注入后正文
- 必须输出引用清单（每条 cite 可追溯）
- 必须给出 .bib 缺失清单（让用户补充真实文献）
- 必须给出查重率预估
- 必须指向 Phase Final
