## 1. 问题陈述

`aigc-reduce-skills` 降 AIGC 流水线为压低 CNKI AIGC 率，引入了过度口语化、过度随意的表达，不符合学术论文语体。用户实测发现的反例：

- `主导权也反转了个个`（破碎语法 + 口语）
- `那么这么做的意义是什么呢？`（反问句）
- `这个结果有点反直觉。`、`为什么有效？`、`数字说明了问题。`、`实际跑下来`、`麻烦在于`（口语碎句）

## 2. 根因定位

口语化表达来自两个机制，并被复制扩散到多个文件：

| 来源机制 | 产物 | 所在文件 |
|---|---|---|
| 思维痕迹注入词库（Stage 5） | `为什么有效？`、`数字说明了问题。`、`这个结果有点反直觉。`、`实际跑下来`、`麻烦在于`、`根源很可能是` | `aigc-reduce-perplexity` |
| 去结构化改写示例（Stage 0） | `绕不过的坑`、`更棘手的是`、`换个视角看`、第一人称 `我们的切入点是` | `aigc-reduce-destructure` |

扩散点：`aigc-reduce`（编排器，复制了 v3 原则 + 思维痕迹词汇库表）、`aigc-reduce-vocab`（CNKI 安全替代表把 `该现象提示：` 直接改成 `为什么有效？`）、`aigc-reduce-rhythm`（短句植入示例）、`README.md`（流水线表）。`aigc-reduce-cohesion`、`aigc-reduce-hedging`、`aigc-reduce-cite-inject`、`aigc-scan` 基本干净。

**关键认知**：思维痕迹注入是 CNKI 实测验证过的有效降痕手法（实测 5.9% → <1%）。其有效性的本质不是「口语化」，而是「认知内容」——承认反直觉结果、暴露机制疑问、叙述方法取舍。该层内容完全可以用书面学术语体表达。因此本次改造的目标是**改写哲学，而非删除机制**。

## 3. 目标语体（用户已确认）

- **正文章节** = L1 严格书面学术体：零反问句、零口号碎句、零口语词、零中文第一人称。
- **认知深度保留 L2**：反直觉结果、机制疑问、方法取舍等「研究过程」事实必须保留，但以客观陈述句承载。
- **英文 `we` 保留**：英文学术论文使用 `we` 是规范，不在禁用范围；禁用仅针对中文 `我们/课题组`。
- **致谢节豁免**：致谢（Acknowledgement）按惯例是个人化谢词，保留第一人称与现有 `aigc-reduce-destructure` 致谢改写示例，不受本规范红线约束。

## 4. 方案：共享语体规范 + 外科修改（方案 B）

新增一个权威文件 `_REGISTER-SPEC.md` 作为单一语体真相源；各产出正文的 skill 把口语词库/示例替换为 L1 版本，并加一行指针引用该规范。不改动流水线 Stage 编号、skill `name`/触发词、`detect_aigc.py`。

### 4.1 新增 `_REGISTER-SPEC.md`

路径：`Paper_Writing/aigc-reduce-skills/_REGISTER-SPEC.md`。内容五块：

#### ① 语体定级
正文 = L1 严格书面体；保留 L2 认知内容；致谢豁免。（同第 3 节。）

#### ② 四条禁用红线（正文，致谢除外）
1. **禁反问句** —— 例：`为什么有效？`、`这么做的意义是什么呢？`
2. **禁口号式碎句** —— 例：`数字说明了问题。`、`结果出乎意料。`。短句仍允许（节奏变频需要句长方差），但必须是完整、有学术内容的陈述句。
3. **禁口语词** —— 例：`有点`、`绕不过的坑`、`实际跑下来`、`跑通之后`、`麻烦在于`、`换个视角看`、`反转了个个`、`说不了太多`。
4. **禁中文第一人称** —— `我们`、`咱们`、`课题组`（作主语时）。英文 `we` 保留。

#### ③ 口语→L1 映射表（核心）

思维痕迹（覆盖 `aigc-reduce-perplexity` 全部 14 条）：

| 口语原版 | L1 客观书面版 |
|---|---|
| 这个结果有点反直觉。 | 该结果与预期不符。/ 一个出乎预期的现象是，…… |
| 数字说明了问题。 | 上述数据已清楚反映这一点。 |
| 为什么有效？ | 其有效性可解释如下。/ 该机制的作用路径分析如下。 |
| 从物理上看…… | 从物理机制层面分析，…… |
| 换句话说…… | 亦即，…… / 更准确地说，…… |
| 没有走……这条路。 | 本文未采用……路径。 |
| 起初尝试了……，但发现…… | 前期实验中曾尝试……，结果表明…… |
| 实际跑下来…… | 实际运行结果显示，…… |
| 跑通之后才发现…… | 系统集成后发现，…… |
| 麻烦在于…… | 其主要困难在于…… |
| 难点集中在…… | 难点主要集中在…… |
| 根源很可能是…… | 造成这一现象的主要原因在于……/ 其根源可归结为…… |
| 这从侧面佐证了…… | 这一现象间接印证了…… |
| 这正是……的出发点。 | 这构成了……的设计动因。/ 这一问题正是……的提出动机。 |

去结构化口语词（覆盖 `aigc-reduce-destructure` 改写示例）：

| 口语原版 | L1 客观书面版 |
|---|---|
| 绕不过的坑 | 难以回避的问题 |
| 更棘手的是 | 更为困难的是 / 更突出的问题是 |
| 换个视角看 | 从另一角度分析 |
| 这条路确实丢掉了 | 该方案确实牺牲了 |
| 另一个短板 | 另一处不足 |
| 我们的切入点是 | 本文的切入点是 |
| 其实有几个…… | 存在若干…… |

#### ④ 主语池
正文主语轮换池 = `本文 / 本课题 / 本章 / 该方法 / 上述策略 / 该现象 / 无主语被动`（如「实验结果显示」「经验证」）。**删除旧池中的 `我们 / 课题组`**。

`本文/本课题 密度 < 30%` 指标保留（连续多句以 `本文` 开头确被 CNKI 标红），但降密度手法从「换成我们」改为「换成客观替代主语 + 无主语化」。

#### ⑤ 致谢豁免条款
明确声明：以上四条红线与主语池不适用于致谢节。致谢继续按 `aigc-reduce-destructure` 现有致谢改写示例（个人化、口语化语气）处理。

### 4.3 思维痕迹重铸示例（`aigc-reduce-perplexity` 改写示例新版）

替换该 skill 第 99-112 行「改写示例」段落。

**原文（纯 AI 语气，作为反例保留展示）**：
> 该现象提示：特征工程中盲目堆叠通道并非更优，Ch2 引入的噪声拖累了整体性能。

**新 L1 版（取代当前口语版）**：
> 该结果与预期不符：引入 Ch2 通道后识别精度不升反降。造成这一现象的主要原因在于，在当前信噪比条件下，Ch2 引入的相位噪声覆盖了 Ch3 所携带的判别信号。这一现象间接印证了 Ch3 的实际贡献，同时暴露出一处工程隐患：在缺乏约束的情况下，网络倾向于优先利用幅度通道而弱化相位通道。这一问题构成了下一小节幅度 Dropout 策略的设计动因。

特征：反检测的「研究过程」信号全部保留（反直觉结果、根因分析、机制印证、设计动因），但零反问、零碎句、零口语、零第一人称、零破折号。

- 不合并/拆分任何 skill（方案 C 已否决）。
- 不改 `detect_aigc.py`、`aigc_reducer.py`、`requirements.txt`。
- 不改 `aigc-reduce-destructure` 的致谢改写示例（用户确认致谢维持现状）。
